Engle Docs: Import Specifier

Import specifiers are Engle's way of specifying modules to import, and are written in import notation, which accepts

  1. "quoted/file names" with forward slashes (/) as a path separator, even on Windows, for local modules,
  2. single_word_names for builtin modules,
  3. <names in angle brackets> for installed modules. The advantages of this syntax are that module names can include spaces and can't conflict with builtin ones.
In addition to this, import notation accepts an optional version selector the end of a specifier, separated by an at sign (@).

Examples

Import Specifier Matches
math the builtin math module
<mcpi> an installed module called mcpi, presumably for the Minecraft: Pi Edition API
<math> an installed module that happens to be called math, rather than the builtin math module
<math>@2.x any version of the second major release of an installed module called math
<math>@2.x-3.x any version of the second or third major release of an installed module called math, presumably because 4.0 broke your project
<math>@3.(2.1)+ any version of the third major release of an installed module called math after 3.2.1, presumably that update fixed a bug that affected your project
"qol.xml" presumably project-specific quality-of-life stuff
