Engle: Directives

Engle borrows a lot from C-related languages like C, including directives. Directives are special commands that are processed separately from the actual program. A directive is written as a single line whose first non-whitespace character is an octothorpe (#). These lines are treated as blank when the program is parsed.

Compiler Directives

#error <text ...>
where text is a "greedy string"
generates a compiler error.
#info <text ...>
where text is a "greedy string"
generates a compiler message.
#warn <text ...>
where text is a "greedy string"
generates a compiler warning.
#include <module> as <alias>
where module is an import specifier
and alias is a valid unused namespace
includes a module using an alternate name.
#include <module>
where module is an import specifier
includes a module in your program.
#silence
disables debug logging for your program.
#silence <module>
where namespace is a valid namespace
disables debug logging for a specific module.
